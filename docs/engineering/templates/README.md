# SEAM Engineering Templates

Copy the relevant template into the issue, audit, branch notes, or PR body. Remove unused prompts only after answering why they are not applicable.

- [Change plan](CHANGE_PLAN.md) — request, change classes, current-state trace, hypothesis, baseline, security/failure analysis, and verification plan for a change.
- [Architecture decision (ADR)](ARCHITECTURE_DECISION.md) — context, decision, alternatives, data/control-flow and security impact, migration, acceptance, and supersession.
- [Threat-model delta](THREAT_MODEL_DELTA.md) — changed boundaries, attacker-controlled fields, abuse and partial-failure cases, controls, and residual risk for a security-sensitive change.
- [Engineering handoff](ENGINEERING_HANDOFF.md) — scope, diagnosis, hypothesis/baseline, verification performed, benchmark evidence, security delta, and the first next command.
- [Incident report](INCIDENT_REPORT.md) — severity, affected scope, timeline, root cause, containment, eradication/recovery, evidence, corrective actions, and continuity updates.
